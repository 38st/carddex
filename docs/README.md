# Carddex docs

Planning artifacts for bringing Carddex to the App Store. Produced by a four-role planning pass
(product designer, iOS engineer, visual designer, backend engineer) and synthesized into a launch plan.

- [launch-plan.md](launch-plan.md) — the unified, synthesized plan (start here)
- [product-plan.md](product-plan.md) — personas, scope, monetization, positioning, roadmap
- [design-spec.md](design-spec.md) — "The Vault" visual & interaction spec, token upgrade map
- [ios-engineering-plan.md](ios-engineering-plan.md) — architecture, camera, identification, sync, CI, App Store reqs
- [backend-plan.md](backend-plan.md) — Supabase schema, Edge Functions, identification service, API contract

These are living documents — update them as decisions change.
